# Custom Executing Plans

## 原則

計画ドキュメントが唯一の指示書である。書かれたことだけを、書かれた順序で実行する。

## プロセス

### 1. 計画を読む

1. 計画ドキュメントを通読し、全体像を把握する
2. 不明点・矛盾・前提条件の不一致があれば、実行前に質問する
3. 問題がなければ実行を開始する

### 2. ステップを順に実行する

1. 計画に記載された順序でステップを実行する
2. 各ステップで検証手順が指定されていれば、必ず実行する
3. 検証に失敗した場合は停止し、原因を報告する

### 3. 完了を報告する

全ステップ完了後、実行結果と検証結果を報告する。

## やらないこと

- 計画にないリファクタリング
- 計画にないファイルの作成・変更
- 計画のステップ順序の変更
- 検証ステップの省略
- 計画にない「改善」の追加

## 停止条件

以下の場合は実行を中断し、質問する:

- 手順が不明瞭で解釈が分かれる
- 指定されたファイルやリソースが存在しない
- 検証が繰り返し失敗する
- 計画の前提が現状と合っていない
