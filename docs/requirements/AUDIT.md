# 監査チーム (Audit Team) 仕様

## 基本構造

- **プロジェクトと同様の構造**
- 監査用エージェントを割り当てたプロジェクト（またはチーム）

```
[プロジェクトA]              [監査チーム]
 ├─ エージェント1             ├─ 監査エージェント1
 ├─ エージェント2    ←監視──  └─ 監査エージェント2
 └─ エージェント3
```

---

## 監査エージェントの特徴

### 強いアクセス権限
- 全プロジェクトの閲覧権限
- 全タスク・履歴へのアクセス
- エージェントの行動ログ閲覧

### ロック権限
- タスクの強制ロック（作業停止）
- エージェントの強制ロック（活動停止）
- プロジェクトの強制ロック？

---

## 位置づけ

- プロジェクトと類似だが**別概念**として実装
- 将来的に異なる派生の可能性を考慮

```
[トップレベル]
 ├─ プロジェクト群
 ├─ エージェント群
 └─ 監査チーム群 ← 別概念
```

---

## ロック機能

### ロックの種類
| 種類 | 対象 | 効果 |
|------|------|------|
| タスクロック | 特定タスク | 状態変更を禁止 |
| エージェントロック | 特定エージェント | 全操作を禁止 |

### ロック解除
- **監査エージェントのみ**解除可能
