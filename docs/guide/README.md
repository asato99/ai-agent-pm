## 基本原則

1. **クリーンアーキテクチャ**: 依存関係は外側から内側へのみ
2. **TDD必須**: テストなしでプロダクションコードを書かない
3. **DI必須**: コンストラクタインジェクションでテスト可能に
4. **命名一貫性**: 規則に従った命名で可読性を確保
