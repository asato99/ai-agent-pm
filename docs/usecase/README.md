## ドキュメント構成

各ユースケースは以下の構成で記述:

1. **概要** - ユースケースの目的
2. **前提条件** - 実行前に必要な状態
3. **アクター** - 関与する人/システム
4. **基本フロー** - 正常系の手順
5. **シーケンス図** - アクター間のやり取り
6. **ステータス遷移** - 状態変化
7. **関連エンティティ** - 操作対象
8. **備考** - 補足事項
